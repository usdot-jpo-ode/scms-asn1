60 -- @brief The PolicyFiles structure defines the parent structure for all
61 -- policy files (GCCF & LCCF). Each policy file resides in its own file
62 -- and its signed by one or more components. to ensure the policy is
63 -- valid.
64 -- @class PolicyFiles
65 -- @param globalPolicyFile contains the global policy file generated by Policy
66 -- Generator (PG).
67 -- @param localPolicyFile contains the local policy file genrated by a
68 -- Registration Authority (RA). Note that RA has to
69 -- get this signed by PG before sending to EEs.
70 PolicyFiles ::= CHOICE {
71  globalPolicyFile GlobalPolicyFile,
72  localPolicyFile LocalPolicyFile,
73  ...
74 }

77 -- @brief This data type defines the inherent policy file structure created
78 -- either by PG or RA.
79 -- @class BasePolicyFile
80 -- @param version defines the version of BasePolicyFile. Currently, it is
81 -- denoted by integer 1.
82 -- @param tbsData is the policy data that is signed by PG at the scms
83 -- protocol level.
84 -- @param signatures denote the counter signatures that are generated by
85 -- auditors of the policy file. Note that PG or RA must
86 -- obtain these signatures before sending to any EE.
87 -- @see Uint8, Countersignature
88 BasePolicyFile ::= SEQUENCE {
89  version Uint8(1),
90  tbsData ToBeSignedPolicyData,
91  -- countersignatures generated by auditors of the policy file
92  signatures SEQUENCE SIZE(1..MAX) OF Countersignature,
93  ...
94 }
